# 客户配置 {#customer-configuration}

“客户配置仪表板”是一款强大的工具，可为您提供品牌在 LLM 中的可见度洞察。 通过正确设置类别、主题和提示词，您可以确保品牌在 LLM 生成的回答中占据有利位置并更容易得以呈现。 此设置可确保平台根据您的业务情境定制洞察，从而实现对可见度、流量和机会的准确分析。

![客户配置仪表板](/help/dashboards/assets/customer-config.png)

要配置 LLM Optimizer 如何在不同市场和竞争格局中监测并分析您的品牌存在感情况，您可以使用以下选项卡：

* [提示词](#prompts-brand)
* [类别](#categories)
* [其他品牌](#other-brands)
* [品牌别名](#brand-aliases)
* [内容传递网络配置](#agentic-cdn)
* [Google Search Console](#google-console)

>[!IMPORTANT]
>
> 有关如何设置类别、主题和提示词的更多信息，请参阅[配置类别、主题和提示词的最佳做法](/help/overview/best-practices-topics-prompts.md)页面。

## 提示词 {#prompts-brand}

在此选项卡中，您可以查看、管理并自定义提示词。 您可以上传[品牌存在感分析](/help/dashboards/brand-presence.md)的 .csv 文件，该列表中将填充该分析中的提示和主题，或者[下载 Adobe 创建的提示库](/help/overview/best-practices-topics-prompts.md)。 您也可以按需删除、修改或添加主题及其关联的提示词。

要导入数据洞察 .csv 文件，您需要先从品牌存在感仪表板导出文件。 请参阅[数据洞察](/help/dashboards/brand-presence.md#data-insights)部分了解具体操作。 获取文件后：

1. 在仪表板中，单击&#x200B;**上传 CSV**。
2. 在“导入数据洞察”窗口中，拖放文件或手动选择文件。
3. 单击&#x200B;**上传数据**。

您也可以从&#x200B;**导入数据洞察**&#x200B;窗口下载模板，以创建新的 CSV 文件。 获取模板后，请打开模板，并将主题及其关联的提示词、类别和区域按“每行一条”的方式录入。

要了解如何下载并使用 Adobe 创建的行业提示词库，请参阅[此页面](/help/overview/best-practices-topics-prompts.md)中的“行业提示词库”部分。

此外，您也可以不依赖 CSV 文件或提示词库，直接将主题/提示词添加到列表中。 要执行此操作，请在仪表板中：

1. 单击&#x200B;**添加主题**&#x200B;按钮。
2. 在新的配置窗口中，选择&#x200B;**类别**。 此前创建的类别将显示在此处。
3. 输入主题名称。
4. 添加提示词文本。
5. 选择区域。
6. 单击&#x200B;**添加提示词**&#x200B;后，包含该提示词的主题将显示在列表中。

>[!NOTE]
>在完成处理之前，新添加的提示词不会在品牌存在感中显示。

在列表中，您可以单击任意主题以查看其关联的提示词。要删除该主题及其关联提示词，请单击列表中的删除图标。

## 类别 {#categories}

在类别选项卡中，您可以定义希望跟踪的业务类别或产品线，并将其关联到特定区域。 总体而言，类别选项卡与本页面上的几乎所有其他自定义设置相关，因为类别会出现在其他配置（如其他跟踪、别名等）的类别字段中。 要添加新类别：

1. 单击&#x200B;**添加**&#x200B;按钮。
2. 在新的配置窗口中，输入&#x200B;**类别名称**。
3. 自定义要监测该类别的&#x200B;**关联区域**。
4. 单击&#x200B;**保存**，新类别将显示在类别列表中。

添加新类别不会自动生成主题和提示词，您需要在[数据洞察](#data-insights)选项卡中手动添加。

要删除类别，请在类别列表中单击删除图标。 请注意，**删除类别将同时删除与该类别关联的项目**，例如链接到该类别的品牌别名。

## 其他品牌 {#others-tracking}

通过此选项卡，您可以跟踪其他品牌在不同类别和区域中与您的品牌相关的提及情况。 监测它们在您的市场细分中的呈现情况和表现。 要自定义跟踪：

1. 单击&#x200B;**添加**&#x200B;按钮。
2. 在新的配置窗口中，选择&#x200B;**类别**。 此前创建的类别将显示在此处。
3. 输入其他品牌名称。
4. 如有需要，自定义该品牌的别名和域名。
5. 单击&#x200B;**保存**。

要删除列表中的条目，请单击删除图标。

## 品牌别名 {#brand-aliases}

通过使用品牌别名，您可以配置需要在不同类别和区域中跟踪的品牌替代名称和变体形式。 这可确保对所有品牌提及进行全面监测。 要添加品牌别名：

1. 单击&#x200B;**添加**&#x200B;按钮。
2. 在新的配置窗口中，选择&#x200B;**类别**。 此前创建的类别将显示在此处。
3. 选择将会监测该别名的&#x200B;**区域**。
4. 输入品牌别名。
5. 单击&#x200B;**保存**，品牌别名将显示在列表中。

要删除品牌别名，请单击别名列表中的&#x200B;**删除**&#x200B;图标。

## 内容传递网络配置 {#cdn-configuration}

在此选项卡中，您可以配置您的内容传递网络流，以便 Adobe LLM Optimizer 分析您的内容传递网络数据。 这些数据将用于驱动仪表板（例如代理式流量），提供有关流量模式、性能量度和优化机会的洞察。 要加入您的内容传递网络提供商，请单击&#x200B;**加入内容传递网络**。

![客户配置内容传递网络](/help/overview/assets/cc-cdn.png)

在&#x200B;**加入内容传递网络提供商**&#x200B;窗口中：

1. 选择您的内容传递网络提供商。
2. 单击&#x200B;**加入**&#x200B;以启用日志转发。

如果您选择&#x200B;**其他**，则需要联系 llmo-now@adobe.com 以获取支持。

## Google Search Console {#google-console}

Adobe LLM Optimizer允许您集成Google Search Console帐户，将真正的搜索查询直接引入界面。 通过显示真实的Google Search Console查询，您可以构建基于实际搜索行为和高意图发现模式的提示集。 这有助于您根据经验证的需求确定提示的优先级，并将LLM优化工作与用户当前搜索的方式保持一致。 此外，您仍然处于完全控制状态，因为查询永远不会自动添加，并且必须在成为活动提示之前显式选择。

### 工作原理 {#how-it-works}

关于LLM Optimizer与Google搜索控制台之间的集成，要记住的主要事项如下：我们不手动猜测客户可能向AI助手询问什么，而是查看他们&#x200B;**已在搜索的内容**，并将这些真正的查询转换为自然、对话式的提示。 下图举例说明了从搜索查询移动到AI提示的过程。

![进程流](/help/dashboards/assets/diagram-flow.png)

一般来说，该过程包括五个步骤：

#### 步骤1 — 收集实际的搜索数据 {#gsc-one}

该过程以受众通过Google找到您的网站时实际使用的关键字开始。 此原始数据集（通常包含数千个唯一查询）是后续所有操作的基础。

#### 第2步 — 分析含义并筛选安全性 {#gsc-two}

系统会分析每个查询的语义含义（用户真正要问的内容），并通过安全过滤器进行筛选，以删除不适当或品牌外内容。 这确保只向前移动干净的相关关键词。

#### 第3步 — 分组到类别和主题中 {#gsc-three}

相关查询会自动分组为&#x200B;**类别**（宽泛的业务主题）和&#x200B;**主题**（每个类别中重点的子主题）。 系统会优先处理已在LLM Optimizer设置中配置的类别。 此外，它还可能会显示搜索数据显示但尚未监视的新类别。 下图是家具品牌的类别和主题示例：

![家具品牌](/help/dashboards/assets/diagram-example.png)

#### 步骤4 — 以实际关键字为基础生成提示 {#gsc-four}

对于每个主题，系统都会生成类似于真实人与AI助理对话的提示。 每个提示都直接受Google Search Console中实际搜索关键词的影响，从而将关键词意图转换为自然的对话问题。

这种方法（基于关键字）意味着：

* 提示反映的是真实的需求，而不是假想的问题。
* 语言反映了客户实际用词的方式。
* 覆盖范围涵盖用户在您的网站上搜索的所有内容。

提示生成过程还会考虑您的品牌概况（包括产品、竞争对手、行业定位和目标受众），以确保提示在上下文上是准确的。

#### 第5步 — 质量保证和交付 {#gsc-five}

在交付之前，每个提示都会经过多次自动质量检查：

* 重复数据删除 — 删除几乎相同的提示。
* 品牌比例平衡 — 确保现实的混合比例（约75%的非品牌比例，约25%的品牌比例）。
* 语言质量 — 去除机器人短语，以提示声音自然。
* 一致性检查 — 验证日期，删除填充短语，确保简洁的长度。

此外，每个提示都标有其类别、主题、意图类型和品牌/非品牌分类，以便LLM Optimizer开始监控。

#### 提示解剖学 {#prompt-anatomy}

完成上述过程后，交付给LLM Optimizer的每个提示都具有以下属性：

| 字段 | 描述 |
|---------|----------|
| 文本 | 提示符，类似于用户将其键入人工智能助手的方式 |
| 类别 | 分配给此提示的宽泛业务主题。 |
| 主题 | 类别中的特定子主题。 |
| 区域 | 目标市场（例如美国、英国等）。 |
| 意图 | 用户心态：信息性、比较性、事务性、指导性、规划或委派。 |
| 类型 | 类型可以是品牌（提及品牌/产品）或非品牌（一般行业问题）。 |

### 使用方法 {#how-to-use}

按照下面介绍的步骤将Google Search Console查询与LLM Optimizer集成和使用。

#### 连接Google Search控制台 {#connect-console}

在使用此功能之前，您需要将Google Search Console帐户与LLM优化器集成。

1. 打开客户配置功能板。
1. 导航到“Google搜索控制台”选项卡，然后单击&#x200B;**连接帐户**。
   ![Google搜索控制台](/help/dashboards/assets/google-console.png)
1. 使用有权访问所需Search Console资产的Google帐户登录。
   ![Google帐户](/help/dashboards/assets/google-account.png)
1. 选择要连接的属性。
   ![控制台属性](/help/dashboards/assets/console-property.png)
1. 连接完成后，LLM Optimizer开始检索相关的搜索查询。
   ![正在检索数据](/help/dashboards/assets/console-complete.png)

#### 查看和搜索查询 {#search-query}

将Google Search Console帐户与LLM优化程序集成后，您可以查看来自搜索控制台的主题和提示列表，并从列表中添加提示。

1. 在Google Search Console选项卡上，查看源自Search Console的主题和提示列表。
   ![提示列表](/help/dashboards/assets/prompts-list.png)
1. 单击所需的主题/提示类别以展开列表。
1. 使用&#x200B;**添加**&#x200B;按钮从列表中添加提示。 您还可以使用&#x200B;**全部添加**&#x200B;批量添加提示和类别。
   ![添加提示](/help/dashboards/assets/add-prompts.png)
1. 选择满意后，单击通知消息上的&#x200B;**保存**。

#### 查看在提示列表中添加的查询 {#prompts-list}

添加查询后，该查询将显示在“客户配置”仪表板的[提示](#prompts-brand)选项卡中。 来自Google Search Console的提示在&#x200B;**Origin**&#x200B;列中用Google Search Console图标标记。 图标可帮助您区分基于实际用户搜索行为的提示与手动添加或其他来源的提示。

### 常见问题解答 {#gsc-faq}

问：在Google Search Console功能板中更新提示的频率如何？

来自Google Search Console的提示通常每月刷新一次。 每次刷新都会从Google Search Console中提取最新的搜索查询数据，重新运行生成管道，并更新提示符集。 这可确保您的提示与当前搜索趋势以及用户行为的季节性变化保持一致。

问：通常有多少提示源自Google Search Console？

数量取决于部署的大小和跟踪的类别数量。 例如：

| 类别 | 主题总数 | 提示已送达 |
|---------|----------|----------|
| 1-2 | 3-8 | ~65-180 |
| 4-5 | 12-20 | ~270-450 |
| 10 | 30-40 | ~675-900 |

我们的目标是提供符合在试用和入门过程中传达的质量目标的提示集：每个主题至少20个提示，每个类别有3到4个主题，以及健康的品牌/非品牌平衡。

问：连接到Google Search Console后，我多久才会看到源自Google Search Console的提示？

建立Google Search Console连接后，通常在&#x200B;**几小时**&#x200B;内有提示可用。 该管道会自动提取您的搜索数据，在生成和质量保证步骤中对其进行处理，并将最终的提示集发送给LLM Optimizer。

问：谁可以连接到Google Search Console？

在Google Search Console属性上具有&#x200B;**所有者**&#x200B;或&#x200B;**完全权限**&#x200B;的任何人都可以授权连接。 这些权限级别授予对搜索查询数据的读取权限。 如果您不确定自己的权限级别，可以在Google Search Console的&#x200B;**设置>用户**&#x200B;和权限下查看该级别。

问：能否将提示标记为已忽略或已跳过，以便在Google Search Console提示列表中看不到它们？

可以，您可以删除不想监视的任何提示。 已删除的提示将从活动提示列表中删除，并且不会显示在将来的报告中。 如果删除的提示在随后的每月刷新中重新生成，您可以再次删除它。

问：一旦我从Google Search Console向提示列表添加提示，我何时会看到这些提示的品牌存在感数据？

新添加提示的品牌存在感数据将在下次计划数据刷新期间显示，该刷新通常在每周初运行。 根据添加提示的时间，您可能会在几天内看到结果。
