# 添加目录

>[!NOTE]
>
> **提前访问** — 此功能在提前访问中可用。 随着功能的成熟，工作流的可用性、资格和部分可能会发生变化。 如果您对访问权限存有任何疑问，请联系您的Adobe客户团队。

添加目录机会标识缺少清晰的&#x200B;**目录**&#x200B;和结构指南的高流量页面，这会使AI代理更难分析页面并将用户查询映射到正确的部分。 它引入了一个结构化的目录，该目录具有&#x200B;**锚点链接标题**，反映页面的主要部分。 这种结构有助于代理更可靠地提取、映射和引用相关段落。

对于每个受影响的URL，您可以查看建议的目录条目，然后通过[在Edge中优化](/help/dashboards/optimize-at-edge/overview.md)来部署这些条目，这样代理流量就可以获得更清晰的导航上下文，而无需更改内容管理系统(CMS)。

## 它如何修复问题

使用Edge[&#128279;](/help/dashboards/optimize-at-edge/overview.md)中的优化来应用修复，其中：

- 向AI代理提供预渲染的HTML快照。
- 将目录添加到页面中。
- 在CDN层工作（CMS不会发生更改）。
- 是仅限AI的 — 不会对人类访客或SEO机器人造成影响。
- 几分钟即可部署，并且可从LLM Optimizer界面&#x200B;**完全还原**。

## 工作原理

LLM Optimizer标识高流量页面，其中&#x200B;**目录**&#x200B;将改进AI代理导航标题和区域的方式。 对于每个页面，建议都基于页面上已存在的标题，因此目录反映了页面结构。

受影响的URL出现在&#x200B;**当前建议**&#x200B;选项卡上带有建议&#x200B;**表的** URL中。

在&#x200B;**当前建议**&#x200B;中，对于每个URL，您可以：

- **展开行**&#x200B;以检查建议的目录（从页面上的标题进行解析并以锚点链接条目形式显示）。
- 比较前后&#x200B;**预览**。
- 如果您在LLM Optimizer外部处理了商机，请&#x200B;**标记为已修复**。
- **忽略不相关的**&#x200B;建议。

建议被整理为&#x200B;**当前建议**、**固定建议**&#x200B;和&#x200B;**忽略的建议**，这与其他在Edge中优化的机会。

### 部署优化

准备在边缘发布时，选择要部署的目录建议。 页脚总结了所选项目的数量，通常提供&#x200B;**标记为已固定**、**忽略建议**&#x200B;和&#x200B;**部署优化**。

单击&#x200B;**部署优化**。 **部署到Edge**&#x200B;对话框列出了选定的URL和优化详细信息。 查看列表，然后选择&#x200B;**部署**&#x200B;或&#x200B;**取消**。

成功部署后，**部署完成**&#x200B;将确认有多少优化已上线。 关闭对话框并打开&#x200B;**修复建议**&#x200B;以验证状态。

>[!NOTE]
>
>部署优化需要完成Optimize at Edge载入流程。 如果您尚未载入，单击&#x200B;**部署优化**&#x200B;会将您引导至载入流程。 有关在Edge中优化的工作方式、支持的CDN提供商以及载入流程的完整详细信息，请参阅[在Edge中优化](/help/dashboards/optimize-at-edge/overview.md)页面。

### 修复了建议并实时查看

在&#x200B;**已修复建议**&#x200B;上，部署的URL在状态列中显示&#x200B;**已优化**。 展开一行以查看部署的目录，在可用的情况下使用&#x200B;**详细信息**&#x200B;进行分析，或单击&#x200B;**实时查看**&#x200B;打开作为验证依据的&#x200B;**当前页面内容**&#x200B;的只读视图（包括插入的&#x200B;**目录**）。

当需要批量还原边缘更改时，请使用复选框选择优化的行，然后在标题中使用&#x200B;**Rollback**。

## 回滚

如果您改变主意，可以回退已部署的优化。 从&#x200B;**修复建议**&#x200B;视图中，选择要还原的优化行，然后单击标题中的&#x200B;**回滚**。

**Rollback**&#x200B;对话框列出了将回滚的建议，并简短地警告将还原已部署的优化。 确认列表，然后单击&#x200B;**回滚**&#x200B;或&#x200B;**取消**。

操作完成后，将显示&#x200B;**已成功回滚**&#x200B;摘要；关闭它以返回到仪表板。
