# 代理式流量 {#agentic-traffic}

“代理式流量”仪表板展示 AI 代理（爬虫和聊天机器人）与您的站点交互的情况。使用此视图，您可以跟踪请求总数以及常规的与性能相关的量度。您还可以查看流量在市场、类别、页面和代理之间的分布情况。此仪表板使用的数据来自内容传递网络日志，因此您需要配置&#x200B;**内容传递网络日志转发**，仪表板才会显示相关量度。此外还提供可自定义的筛选条件，帮助您细化要显示的数据。

## 内容传递网络日志转发 {#cdn-setup}

未配置&#x200B;**内容传递网络日志转发**&#x200B;时，“代理式流量”仪表板不会显示任何内容。要查看代理交互数据，您需要配置&#x200B;**内容传递网络日志转发**。首次登录时，您会看到如下图所示的提示信息。

![内容传递网络设置](/help/dashboards/assets/ag-log-forward1.png) 

选择&#x200B;**前往配置**&#x200B;后，系统会自动跳转到[客户配置仪表板](/help/dashboards/customer-configuration.md)的&#x200B;**内容传递网络配置**&#x200B;选项卡。

![内容传递网络设置：加入](/help/dashboards/assets/ag-log-forward2.png)

在该选项卡中，选择&#x200B;**加入内容传递网络**。随后会显示内容传递网络提供商窗口。

<!-- [CDN Provider](/help/dashboards/assets/ag-log-forward3.png)-->在&#x200B;**加入内容传递网络提供商**&#x200B;窗口中：

1. 选择您的内容传递网络提供商（例如 Akamai、Adobe 管理的 Fastly、Fastly、AWS Cloudfront、Azure CDN、Cloudflare 或“其他”）。
2. 单击&#x200B;**加入**&#x200B;以启用日志转发。

如果您选择&#x200B;**其他**，则需要联系 llmo-now@adobe.com 以获取支持。

启用后，系统会开始摄取日志，仪表板将显示总代理交互次数、成功率、按市场划分的点击数、用户代理分析以及 URL 级别性能等量度。

LLM Optimizer 会处理内容传递网络日志中的一部分字段。不同内容传递网络提供商的原始日志字段名称可能不同，但系统会将其规范化并以以下形式呈现：

* URL（仅路径）
* 用户代理
* 状态代码
* 引荐标头
* 主机标头
* 首字节时间（TTFB）
* 请求方法
* 时间戳
* 内容类型

代理视图会展示这些规范化后的字段。在[引荐流量](/help/dashboards/referral-traffic.md)仪表板中，系统会使用内容传递网络日志来展示页面点击量度。在内容传递网络日志摄取及后续数据处理的任何阶段，系统都不会处理或存储任何个人身份信息（PII）。

## 筛选条件 {#filters}

在页面顶部，您可以应用筛选条件以优化视图。您选择的筛选条件将影响仪表板中的&#x200B;**所有**&#x200B;部分。您可以自定义以下内容：

* **日期范围**：选择显示数据的时间范围。例如，最近 4 周。您也可以选择&#x200B;**自定义周数**&#x200B;选项来自定义时间周期。
* **类别**：按预定义类别或自定义类别筛选显示结果。
* **平台**：选择要分析的 AI 引擎。
* **代理类型**：按与您站点交互的 AI 代理类型进行筛选。您可以选择爬虫、聊天机器人或所有代理。
* **成功率**：按交互质量（高、中或低）进行筛选。该量度表示成功 HTTP 请求的百分比，包括直接成功响应（2xx 状态代码）和重定向（3xx 状态代码）。
* **内容类型**：查看不同内容类型（如 HTML、PDF 等）的代理交互情况。

选择所需筛选条件后，单击&#x200B;**应用筛选条件**&#x200B;以将其应用到仪表板。

## 流量分布 {#traffic-distribution}

“流量分布”视图展示代理式流量在不同市场、类别和页面类型之间的分布情况。通过该视图，您可以了解 AI 代理在与您的站点交互时最常访问的地理区域、产品领域或内容类型。

![流量分布](/help/dashboards/assets/ag-main.png)

页面顶部显示三个您需要注意的关键量度：

* **代理交互次数**：表示 AI 代理向您的网站发出的请求总数。该指标包括来自搜索引擎、聊天机器人及其他非人工流量的所有请求。
* **成功率**：表示成功 HTTP 请求的百分比，包括直接成功响应和重定向。
* **平均 TTFB**：首字节时间（TTFB）表示从服务器接收到第一个数据字节所需的时间。平均值根据各状态代码对应的请求数量进行加权计算，并排除返回 5xx 状态代码的请求。数值越低表示服务器响应速度越快。

每个关键量度都配有趋势指示器，用于显示与上一周期相比的变化情况。

## 代理式流量趋势 {#agentic-trends}

使用“代理式流量趋势”图表可以跟踪每周的成功请求、失败请求和总点击数。通过该图表，您可以随时间监测代理活动与性能的变化。您还可以将鼠标悬停在图表上，查看每周时间框架内的数据变化情况。

![代理式流量趋势](/help/dashboards/assets/ag-trends.png)

## 涨跌幅最大页面 {#top-bottom-movers}

“涨跌幅最大页面”视图会突出显示代理式流量周环比变化最大的 URL —— 即来自 AI 系统访问您内容的访问量或点击数。**涨幅最大**&#x200B;显示可见度或参与度提升的页面，**跌幅最大**&#x200B;显示下降幅度最大的 URL。通过该视图，您可以快速识别哪些内容正在呈现上升趋势、哪些内容需要关注，以及 AI 驱动的发现模式出现了哪些变化。

![涨跌幅最大页面](/help/dashboards/assets/movers.png)

## 用户代理与 URL 性能分析 {#user-url-performance}

“用户代理与 URL 性能分析”视图对爬虫和聊天机器人与您站点的交互情况进行更深入的数据拆解。点击下方选项卡可查看详细说明。

![用户代理与 URL 性能分析](/help/dashboards/assets/user-agent.png)

>[!BEGINTABS]

>[!TAB 用户代理分析]

“用户代理分析”表按页面类型和代理类型（例如爬虫与聊天机器人）对流量进行细分。通过该表，您可以清楚了解不同 AI 代理访问您网站的哪些部分。该表包含以下类别：

* **页面类型**：页面类型。
* **代理类型**：访问该页面的 AI 代理类型（爬虫或聊天机器人）。
* **点击数**：AI 代理针对该页面类型发出的请求总数。

点击&#x200B;**配置列**&#x200B;按钮，您可以自定义要显示的量度。

>[!TAB URL 性能分析]

“URL 性能分析”表展示各个 URL 的详细数据视图。其中包括点击数、独特代理数量、主要代理、成功率和类别等。通过该表，您可以识别高价值页面、发现抓取覆盖缺口，并针对 AI 引擎优化内容。URL 按流量大小排序。该表包含以下类别：

* **URL**：检查的 URL。
* **总点击数**：AI 代理向该 URL 发出的请求总数。
* **独特代理数**：访问该 URL 的不同 AI 代理数量。
* **主要代理**：为该 URL 带来最多流量的 AI 代理。
* **主要代理类型**：为该 URL 带来最多流量的 AI 代理类型。
* **成功率**：成功 HTTP 请求的百分比，包括直接成功响应和重定向。
* **类别**：与页面内容最匹配的类别。
* **平均 TTFB（毫秒）**：首字节时间（TTFB）表示从服务器接收到第一个数据字节所需的时间（以毫秒计）。平均值根据各状态代码对应的请求数量进行加权计算，并排除返回 5xx 状态代码的请求。数值越低表示服务器响应速度越快。
* **响应代码**：该 URL 的 HTTP 状态代码。

“URL 性能分析”表提供搜索字段以便快速查找 URL，您还可以通过点击&#x200B;**配置列**&#x200B;按钮来自定义显示的量度。您还可以点击每行末尾的&#x200B;**详细信息**&#x200B;图标，查看每个 URL 的更多详细信息。

![URL 详细信息](/help/dashboards/assets/details.png)

“URL 详细信息”视图提供页面性能的整体概览，包括引用频率、提及该页面的 AI 回答的情绪、出现的主题与提示词，以及代理式流量与引荐流量的随时间变化的趋势。

>[!ENDTABS]

在这两个表格中，您都可以使用&#x200B;**导出**&#x200B;选项下载 `.csv` 文件，与团队共享洞察，或将表格纳入管理层报告。
